# Contributing

Thank you for contributing to this repository.

## Ways to Contribute

You can help by:

- adding new notes
- improving explanations
- fixing grammar or formatting
- suggesting useful resources
- organizing content better

## Contribution Rules

- keep content clear and useful
- write in your own words
- do not post flags
- do not share private lab answers
- do not leak paid content
- keep everything ethical and legal

## Suggested Topics

- enumeration
- pentesting methodology
- red teaming concepts
- security analyst workflows
- web application security
- Linux and networking
- detection and logging

## Pull Request Tips

If contributing through a pull request:

1. explain what you added
2. keep changes focused
3. make sure formatting is clean
4. keep notes beginner-friendly or clearly label advanced content
